## 4. Symfony backend scaffolded

### 4.1 Create Symfony skeleton

```bash
cd bookingSystem-clean-architecture
/opt/homebrew/opt/php@8.4/bin/php /opt/homebrew/bin/composer create-project symfony/skeleton:"7.2.*" backend --no-interaction
```

**Note:** Composer initially blocked some Symfony packages due to security advisories. Resolved with:

```bash
cd backend
composer config policy.advisories.block false
composer update --no-interaction
```

### 4.2 Install backend packages

```bash
cd backend
composer require symfony/orm-pack doctrine/doctrine-migrations-bundle symfony/validator symfony/serializer symfony/property-access --no-interaction
composer require --dev symfony/maker-bundle phpunit/phpunit symfony/phpunit-bridge --no-interaction
```

**Installed stack:**

- Symfony 7.2 (framework, console, routing, etc.)
- Doctrine ORM 3 + DBAL 4 + Migrations
- Validator, Serializer
- PHPUnit 13, MakerBundle (dev)

## 6. Docker install (first run)

```bash
./infrastructure/dev.sh install
```

**What `install` does:**

1. Creates `.env` if missing
2. Creates `backend/.env.local` if missing
3. `docker compose build`
4. `docker compose up -d`
5. Waits for MySQL healthcheck
6. `composer install` inside PHP container
7. Runs migrations if available
8. Prints service URLs and `docker compose ps`

### Port conflicts encountered and fixed

| Attempt | Error | Fix |
|---------|-------|-----|
| Mailpit SMTP `1025` | Port already allocated | `MAILPIT_SMTP_PORT=11026` |
| Mailpit UI `8025` | Port already allocated | `MAILPIT_UI_PORT=18025` |
| MySQL `3306` | Port already allocated | `MYSQL_PORT=13306` |
| API `8080` | Port already allocated | `APP_PORT=18080` |

After port changes, `install` completed successfully.
